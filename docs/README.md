# docs/

Human-readable documentation for the project. Read by developers, reviewers, and AI agents alike.

## Layout

```
docs/
├── CONSTITUTION.md           # Project DNA: what it is, principles, boundaries (optional)
├── CONVENTIONS.md            # Naming, imports, structure conventions (optional)
├── architecture/             # System design
│   └── overview.md           # High-level architecture, trust model, data flows
├── decisions/                # ADRs (immutable, numbered)
│   ├── README.md
│   └── NNNN-title.md
├── runbooks/                 # Operational procedures
│   ├── README.md
│   ├── deploy.md
│   ├── rollback.md
│   └── incident-response.md
├── guides/                   # Onboarding, how-tos, tutorials
│   ├── README.md
│   ├── getting-started.md
│   └── testing.md
└── patterns/                 # "How we solved X" - living examples
    ├── README.md
    └── <pattern-name>.md
```

## What goes here vs `.claude/docs/`

Use this decision:

- **Humans need to read it? → `docs/`**
- **Only agents need it? → `.claude/docs/`**

Concretely:

| Content | Location |
|---|---|
| Architecture overview, ADRs, runbooks, guides | `docs/` |
| Patterns and design decisions worth teaching | `docs/patterns/` |
| Constitution / current state description | `docs/CONSTITUTION.md` |
| Style and naming conventions | `docs/CONVENTIONS.md` |
| How agents should use a specific external lib in this project | `.claude/docs/libs/` |
| AI-specific workflow docs (Superpowers, spec-driven notes) | `.claude/docs/` |

## The generated docs

Some files here are generated by the `analyze-codebase` skill when adopting the template on an existing project:

- `CONSTITUTION.md` - inferred from project structure and existing docs
- `CONVENTIONS.md` - inferred by sampling representative files
- `architecture/overview.md` - inferred from structure and configs
- `patterns/README.md` - set up as an empty scaffold

Generated files always have `TODO` markers where the analysis was uncertain. Human review is expected.

## The hand-written docs

Some files exist only when a human writes them:

- `decisions/` ADRs - always hand-written, immutable once accepted
- `runbooks/` - hand-written by whoever operates the system
- `guides/` - hand-written for onboarding and tutorials
- Individual `patterns/*.md` - one per solved problem worth teaching

## Referenced by agents

The AI agents in this template read `docs/` extensively:

- `codebase-explorer` reads `CONSTITUTION.md`, `architecture/overview.md`, `CONVENTIONS.md`, and `patterns/`
- `explore` skill reads the same before proposing
- `write-spec` links to them from every new spec
- `code-reviewer` checks changes against `CONVENTIONS.md`

Keep them accurate and current.
