# .claude/context/

Persisted context artifacts generated by skills and subagents. Contents are gitignored (varies per developer and per moment). Only this README is versioned.

## Files that live here

- **`repomix-snapshot.md`** - Full codebase packed into a single file by [Repomix](https://github.com/yamadashy/repomix). Generated by `analyze-codebase` skill, refreshed by `refresh-snapshot` skill or automatically by the `codebase-explorer` subagent when stale.
- **`config.json`** - Threshold overrides and settings for this project (file count threshold for Repomix, staleness thresholds, etc). Optional.
- **`last-analyze.log`** - Timestamp of the last `analyze-codebase` run.

## The staleness model

The snapshot header includes metadata:

```
generated_at: 2026-06-23T14:23:00Z
commit_sha: abc123
files_captured: 247
```

The script `.claude/scripts/check-snapshot.sh` compares this metadata against the current git HEAD and returns a JSON verdict:

| Status | Trigger | What happens |
|---|---|---|
| `fresh` | < 5 files changed AND < 3 days old | Snapshot used as-is |
| `stale-mild` | 5-29 files OR 3-13 days | Used, with a note in the report |
| `stale-major` | 30+ files OR 14+ days OR config file changed | Auto-refreshed by codebase-explorer, warned in SessionStart |

Config changes are weighted more heavily (`tsconfig`, `package.json`, `.eslintrc*`, `tailwind.config`, etc) because they signal convention drift.

## Threshold for generating the snapshot

By default, `analyze-codebase` only generates a Repomix snapshot if the project has 100+ files in `src/`. Smaller projects skip the snapshot and rely on grep/glob directly. This threshold is configurable in `.claude/context/config.json`:

```json
{
  "min_files_for_snapshot": 100,
  "fresh_max_files": 4,
  "fresh_max_days": 2,
  "stale_mild_max_files": 29,
  "stale_mild_max_days": 13
}
```

## Why this is gitignored

The snapshot is derived state, not source of truth. It also varies:

- Different developers have different working states
- Regeneration frequency depends on local activity
- Merging two snapshots would produce conflicts without benefit

If you need shared context across the team, commit `docs/CONSTITUTION.md` and `docs/architecture/` instead. Those are curated by humans.
